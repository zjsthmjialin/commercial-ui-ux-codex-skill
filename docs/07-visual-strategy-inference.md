# 视觉策略自动推断模式

## 1. 目标

当用户没有提供品牌规范、参考图、设计稿或明确风格时，Codex 不应停在“请给我参考图”的被动状态，也不应随意套用流行视觉风格。它应根据项目内容自动生成一套合理的界面配色和样式策略，降低用户的设计表达成本。

这项能力的目标是：用户只需要说清楚“做什么产品、给谁用、在哪个平台、完成什么任务”，Codex 就能先给出一个商业上合理、可验证、可迭代的视觉方向。

## 2. 四种视觉来源模式

| 模式 | 触发条件 | 行动原则 |
| --- | --- | --- |
| 客户指定模式 | 用户提供品牌色、Logo、字体、组件库、设计稿或明确风格 | 严格跟随用户指定内容，偏离时说明原因 |
| 参考图模式 | 用户上传参考图、截图、竞品图或明确要求参考某类视觉 | 提取风格特征，不直接复制受保护设计 |
| 设计系统模式 | 项目已有 Ant Design、Material UI、shadcn/ui、Tailwind token、企业组件库等 | 既有系统优先，避免发明新视觉语言 |
| 项目推断模式 | 用户没有提供视觉资料，但提供了产品内容、行业、用户或任务 | 根据项目上下文自动推断视觉策略 |

优先级：客户指定模式 > 参考图模式 > 设计系统模式 > 项目推断模式。若模式冲突，以更具体、更靠近真实项目资产的来源为准。

## 3. 项目推断模式

项目推断模式是本 skill 的默认核心能力。它适用于用户不知道该用什么配色、按钮、表单、表格或整体风格，但已经描述了产品内容的情况。

### 输入信号

Codex 应从请求和项目上下文中提取：

- 行业：SaaS、金融、医疗、电商、教育、开发者工具、内容平台等。
- 目标用户：专业用户、大众用户、管理者、运营、分析师、消费者等。
- 平台：桌面后台、Web 响应式、移动端、桌面客户端、小程序等。
- 核心任务：审批、筛选、购买、分析、创建、协作、监控、配置等。
- 风险等级：低风险浏览、中风险提交、高风险删除/支付/医疗/金融决策等。
- 数据密度：轻量内容、表单流程、表格后台、高密度图表、实时监控等。
- 既有技术和组件：当前仓库、组件库、设计 token、图标库。

如果用户只给出行业或宽泛领域，应先参考 `docs/09-industry-taxonomy.md`，将其归入顶层行业和子行业，再推断 UI 场景与视觉策略。例如“幼儿教育培训”应归入“教育培训机构”，而不是作为孤立顶层行业处理。

### 推断输出

Codex 应生成一份简短但明确的视觉策略，包括：

- 视觉来源：客户指定、参考图、设计系统或项目推断。
- 视觉关键词：例如可信、克制、高密度、行动导向、温和、安全。
- 主色和辅助色：说明为什么适合该场景。
- 背景与 surface：页面背景、卡片、面板、分割线。
- 字体与字阶：标题、正文、辅助文本、数字指标。
- 按钮样式：主按钮、次按钮、危险按钮、禁用和加载状态。
- 表单样式：label、helper、error、focus、disabled、校验反馈。
- 数据展示样式：表格、卡片、图表、标签、状态色。
- 交互状态：hover、active、focus、selected、loading、error、empty。
- 不适合的风格：例如营销 hero、大面积渐变、玻璃拟态、低对比度灰字等。

## 4. 行业推断参考

完整行业分类见 `docs/09-industry-taxonomy.md`。下表只保留最常用的快速参考。

| 行业/场景 | 推荐视觉策略 | 避免 |
| --- | --- | --- |
| B2B SaaS / 企业后台 | 中性色背景、蓝色或品牌色行动、清晰表格、稳定导航、高信息密度 | 营销 hero、过度插画、大面积渐变 |
| 金融 / 风控 / 数据分析 | 深浅中性色、强对比文字、克制强调色、明确风险色、可审计操作 | 花哨动效、低对比度、娱乐化表达 |
| 医疗 / 健康 | 温和背景、清晰层级、安全提示、错误恢复、谨慎语气 | 过强刺激色、未经证实的结论性视觉 |
| 电商 / 增长 | 行动导向主色、商品图优先、优惠和库存状态清晰 | 隐藏成本、虚假稀缺、误导性默认值 |
| 开发者工具 | 高密度布局、等宽辅助、清晰状态、深浅主题可选 | 装饰性视觉压过信息 |
| 移动审批 / 流程 | 单路径、底部主操作、触控友好、确认/驳回安全 | 只依赖 hover、桌面表格直搬移动端 |

## 6. 常见错误

- 只给“高级、现代、科技感”等抽象词，不落到颜色、控件和状态。
- 忽略项目已有组件库，重新生成一套视觉系统。
- 把行业联想当成固定模板，例如所有金融都做深色、所有医疗都做绿色。
- 为了好看而牺牲表格密度、错误提示、禁用状态或移动端可用性。
- 用户未提供参考图时就停下来等待，而不是先给合理默认方案。
